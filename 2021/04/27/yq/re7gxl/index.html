Pull Request 使用如何简化提交历史

2021-04-27 19:30:00
  • git
  • git

通常我们使用 git 的时候，Pull Request (PR) 是一个非常重要的功能对于多人合作开发一个项目来说。
假设我们有一个项目的提交历史看起来如下图：

这里我们能看出上图提交历史还是一条直线，在 94b7b85422c 提交之后有一次合并操作，很清晰的知道仓库的管理操作历史。
反之，如果提交历史有很多线我们很难追溯过往的提交，如下图：

假设当前仓库后续又有 3 次提交在 master 分支

这个时候在功能 test2 分支上进行开发， 后续提交了 2 次在

可是与此同时 master 分支上也有新的 2 次提交 6166603ce32和 e45069850e1如下图：

所以此时正确的操作应该是在创建 PR 之后，应该先 rebase,如果有冲突需要在本地解决冲突后，提交解决冲突的分支，然后再 merge 到目标分支。
